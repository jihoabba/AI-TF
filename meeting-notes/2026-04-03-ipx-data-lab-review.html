01 · 자료 전달
IPX DATA LAB 제안서 공유
IPX에서 DATA LAB 제안서를 보내왔다. 방향성과 데이터베이스 활용이 겹치는 부분이 있으니 내용 보고 의견 달라.
IPX DATA LAB ML 파이프라인 LLMRec + XAI IP 사업 데이터 통합
📄
IPX 제안 골자 IP 사업 데이터 통합 → ML 모델(Apriori, XGBoost, Prophet 등) → LLMRec + XAI 구축. Phase 1 (3~6개월) 데이터 수집·ML·정기보고서, Phase 2 (1~2년) DW 구축·Insight Engine 완성. 내부 인원 2명 + 클라우드 인프라 연 ~200만원 추산.
02 · 검토 의견
현실성 없는 이유 3가지
❌
만들어도 못 쓴다 구성원들이 AI·디지털 데이터에 익숙하지 않음. 인터뷰에서 직접 확인. 좋은 인사이트가 나와도 활용하는 사람이 없으면 의미 없음.
❌
잘못되면 고칠 수가 없다 ML 파이프라인은 구축 후 유지보수·수정·검증할 내부 인력이 없음. 외부 의존도가 높아질수록 지속 운영이 불가능해짐.
❌
우리 비즈니스 구조에 안 맞는다 ML은 반복되는 패턴이 있어야 작동함. 우리는 팬덤 비즈니스라 아티스트 컴백·팝업 일정·팬덤 규모에 따라 매출 패턴이 매번 달라짐. 일반 커머스처럼 크로스셀 패턴도 없음. ML이 학습할 전형적인 메커니즘 자체가 없는 구조.
03 · 우리 방향
독자 진행 결정
✅
실무진이 당장 필요한 기능부터 인터뷰로 팀별 페인포인트를 파악하고, 즉시 쓸 수 있는 것부터 만든다.
✅
파싱 수준으로 시작한다 로컬에서 코드로 데이터 집계 → 숫자만 LLM에 전달. 보안·재현성·자동화 모두 충족하면서 지금 당장 실행 가능. (파싱 = 엑셀 파일을 코드로 열어서 원하는 숫자를 추출하는 것)
✅
구성원 교육을 병행한다 이게 AX TF의 핵심. 툴을 만드는 것만큼 쓸 줄 아는 사람을 만드는 것이 중요하다. 인프라보다 리터러시가 먼저.
04 · 기술 방식 비교
IPX 방식 vs 우리 방식
IPX — ML 파이프라인 우리 — 로컬 파싱 → 숫자만 LLM 전달
보안 사내 인프라에서 처리, 데이터 유출 없음 로컬 처리, 집계된 숫자만 LLM에 전송. 원본 데이터 외부 유출 없음
분석 알고리즘이 데이터에서 직접 학습. 사람이 모르는 패턴도 발견 가능 사람이 질문을 알고 있을 때 유효. 파서가 코드로 집계, 재현성 100%
자동화 Airflow/MLflow로 주기 자동 실행 파일 변경 감지로 자동 실행. 지금 당장 구현 가능
유지보수 모델 드리프트·피처 업데이트·검증 등 전담 인력 필요 코드 수정으로 대응. 내부에서 직접 관리 가능
LLM 역할 마지막 번역 레이어 (ML 결과 → 자연어) 마지막에 "사람이 읽기 좋게 요약"만 담당
착수 시점 데이터 인벤토리·피처 엔지니어링 포함 3~6개월 지금 바로 시작 가능
