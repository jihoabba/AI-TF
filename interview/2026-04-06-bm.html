인터뷰 — BM 황정윤

📋 1차 인터뷰 · 2~3차 예정
날짜 2026년 4월 6일
팀 BM
인터뷰이 황정윤
주제 세일즈 데이터 관리 · 자동화 가능성 탐색
1차 진단 BM팀 데이터 업무 AI TF 진단
#1
팀별 KPI 미통합 — "애매한 건은 꿀먹은 벙어리"
BM·온라인·오프라인·MD 각 팀의 매출 타겟이 따로 놀고, 통합해서 볼 방법이 없음. 성과 판단 기준 부재로 의사결정 공백 발생.
#2
포캐스팅 로직 부재 — 개인 의지치로 숫자 제출
정해진 포캐스팅 규칙이 없어 숫자를 만들어야 할 때 의지치 반영. 재현·검증 불가한 구조.
#3
팝업·테마별 성과 구분자 수동 처리
IP·채널·날짜 기준 분리는 되나, 팝업/테마/프리오더 단위로는 묶이지 않아 매번 수동 가공 필요.
자동화 여지
보안 이슈만 해결되면 최대
데이터 구조는 정형화돼 있고 추출 작업은 단순. 가공·공유 자동화 즉시 가능.
선결 과제
보안 정책 검토 필수
SAP 전환 예정 + 다부서 데이터 통합 → 데이터 접근 권한·보안 정책 확인 후 진행.
연계 부서
BM·온라인·오프라인·MD·물류
모든 팀이 이 데이터에 의존. 1순위 과제로 집중할 근거.
자동화 의지
강함 — 본인이 직접 원함
단순 부탁이 아닌 본인 주도 니즈. 협업 모티베이션 높음.
01 · 업무 현황
세일즈 데이터 허브 역할
📊
전체 매출 데이터 + 재고 관리 데이터 정리·공유 주 업무. 루틴 리포트 + 요청 건 응대를 병행
🔄
일간 · 주간 · 월간 모든 주기 반복 모든 업무가 정기 반복 → 자동화 효과가 바로 체감되는 구조
🔔
재고 리포팅 업무 추가 (복직 후 신규) 육아휴직 복직 후 업무 추가. 더 직관적이고 단순하게 보여주는 형태로 개선하고 싶음.
02 · 데이터 구조
현재 데이터 흐름
각 채널 어드민 → 공통 양식 가공 → 서식·오염 체크 → 대화방 공유
🗄️
현재 각 채널 어드민 → SAP 방식으로 전환 예정 전환 시 데이터 파이프라인 설계 기회. TF 개입 타이밍.
📅
과거 데이터 3~5년 보유 IP별 / 채널별 / 날짜별 분리 가능. 팝업·테마별은 수동 구분자 처리.
⚠️
팝업·테마별 성과는 자동 묶음 불가 팝업 단위 성과 분석 시 매번 수동으로 구분자 추가 → 자동화 우선순위 과제
03 · 성과 판단 구조의 문제
KPI가 통합되지 않아 성과 판단 불가
세일즈 자료가 있지만 BM팀의 매출 포캐스팅, 온라인팀의 매출 타겟, 오프라인팀의 매출 타겟, MD팀의 매출 타겟이 서로 합의되지 않았고, 구분지어서 볼 수 있는 방법도 없다. 누가 봐도 잘 된 건이나 잘 안된 건은 상관 없는데, 애매한 건들에서는 다들 꿀먹은 벙어리가 될 수밖에 없는 구조.
04 · 포캐스팅 로직
정해진 규칙 없이 의지치로 운영
포캐스팅에 대한 로직은 정해진 규칙이 따로 없다. 일단 숫자를 만들어야 되는 상황에는 의지치를 반영하기도 한다.
🎯
과거 데이터 3~5년 → AI 포캐스팅 모델 훈련 가능 IP별·채널별·시즌별 패턴이 충분히 쌓여 있어 데이터 기반 포캐스팅 도입 여건 갖춰짐
05 · 잠재 니즈
원하는 것
📦
재고 리포트 가독성 개선 복직 후 추가된 업무. 현재 형식을 더 직관적이고 단순하게 바꾸고 싶다.
🔗
데이터 통합 — 각 팀 분산 데이터 한곳으로 지금은 팀별로 흩어져 있음. 통합 정리가 어떤 분석보다 우선.
📡
MD·BM 등 타 팀도 직접 조회 가능한 환경 현재 MD팀이 황정윤님께 데이터를 요청하는 구조 → 셀프서비스 조회로 전환 가능
06 · 우선과제 태스크보드
1차 인터뷰 기반 자동화 후보
🔗 전사 세일즈 데이터 통합 전사
  • BM·온라인·오프라인·MD 데이터 단일 소스화
  • 팀별 KPI / 타겟 합의 구조 설계
  • 팝업·테마·프리오더 단위 자동 성과 묶음
📊 데이터 가공·공유 자동화 팀
  • 어드민 → 공통 양식 변환 자동화
  • 서식·오염 체크 자동화
  • 일간·주간·월간 리포트 자동 생성 및 공유
📦 재고 리포트 대시보드 팀
  • 현재 리포트 직관적 시각화 개선
  • MD팀 연계 (재고·발주 상태 셀프 조회)
🎯 데이터 기반 포캐스팅 2~3차
  • 3~5년 과거 데이터 → IP·채널·시즌 패턴 분석
  • 포캐스팅 로직 정립 (의지치 의존 탈피)
  • 경영진 의사결정 지원 리포트
07 · TF 제안 액션

BM팀 × AI TF 협업 방향

🔐
보안 정책 확인 — 선결 과제 SAP 전환·다부서 데이터 통합 전 데이터 접근 권한 범위 확인. 이 단계가 해결되면 자동화 여지 최대.
⚡
Quick Win: 재고 리포트 가독성 개선 복직 후 추가된 업무, 범위 작고 황정윤님 의지 있음. 2차 인터뷰 전 빠른 프로토타입 가능.
🤝
MD팀 연계 공동 과제 MD팀 실시간 판매현황 니즈 ↔ BM팀 데이터 공유 업무가 정확히 맞닿음. 두 팀 동시 수혜 가능.
📋
2차 인터뷰: 데이터 구조 상세 파악 + SAP 전환 일정 확인 어드민별 데이터 형식, 가공 규칙, SAP 전환 타임라인을 구체적으로 파악해야 자동화 설계 가능.
📅
2차·3차 인터뷰 예정 데이터 구조 상세 파악 → SAP 전환 일정 확인 → 자동화 설계 순으로 진행. 1회성이 아닌 단계적 협업.
